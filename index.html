HTML

  • HTML stands for Hyper Text Markup Language.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • h1 - h6 heading tags are used in order of appearance and size desired.

CSS

  • CSS = Cascading Style Sheet
  • A margin indicated how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • 1. Inline CSS: written into the tags on the HTML file and style only that particular element.
  • 2. Internal style sheet: contains CSS rules for the webpage in the head section of the HTML file and apply to only that page.
  • 3. External CSS style sheet: allows developers to keep all CSS rules in separate file, making design changes easier. See 'separation of concerns.'
  • Separation of concerns: key design principle of separating code into different files based on use.
  • CSS Rule contains: Selector, Declaration = Property + Value, and multiple declarations are a Declaration block.
  • CSS box model: contains Margin, Border, Padding, and Content.

Git

  • Git or Git Bash on PC is the same as Terminal on Mac.
  • All commands entered starting with 'git' connect your local repository to the linked github repo.
  • git status: checks what branch we are currently on.
  • git chekcout -b branch-name: creates a new branch and switches to it.
  • git flow: add, commit, and push work to feature branch on remote repo. Then open a pull request, merge changes with the main branch, and close the associated GitHub issue.
  • git add -A: command to stage your changes
  • git commit -m "your message here": commits changes and message to describe changes made
  • git pull origin main: checks the local branch is up to date, especially before pushing up changes
  • git push origin feature/name-here: push up changes to the remote repository

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script. Top to bottom!
  • Just defining a function does not mean it will execute automatically, we must call it in our code.
  • Arrays store a group of data in a variable and the items listed are zero-indexed - the first item in any array has an index of 0, not 1.
